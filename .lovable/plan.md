המטרה: שלא תצטרך שוב לערוך `.env` ידנית או לחפש ערכים בכל התקנה/החלפת מחשב. ה-`COMPANY_ID` נשאר חובה (שעון פר-חברה), אבל אנחנו עוטפים את כל תהליך ההגדרה במתקין אחד שעושה הכול.

## מה ייבנה

### 1. מתקין אינטראקטיבי יחיד (`agent/setup.js`)
סקריפט Node שמחליף את כל העריכה הידנית של `.env`:
- שואל את 4 השדות שמשתנים בין התקנות: `ATTENDANCE_INGEST_TOKEN`, `COMPANY_ID`, `CLOCK_HOST`, `EMPLOYEE_CODE_PREFIX`.
- כל שאר הערכים (`SUPABASE_FUNCTIONS_URL`, `HARD_MIN_DATE`, `BATCH_SIZE`, `POLL_INTERVAL_MS`, פורטים וכו׳) נכנסים אוטומטית עם ברירות מחדל קבועות.
- אם כבר קיים `.env` — טוען אותו ומציע לשמור את הערכים הקיימים (Enter מאשר).
- מאמת חיבור לשעון (`CLOCK_HOST:4370`) ומאמת ש-`COMPANY_ID` קיים בענן לפני שמירה.
- כותב `.env` תקין, ואז שואל "להתקין כ-Service?" — ואם כן מריץ `service:install` אוטומטית.

### 2. פקודה אחת לכל תהליך ההתקנה
`npm run setup` — מריץ את `setup.js`. זו הפקודה היחידה שצריך לזכור.

### 3. רשימת חברות מהענן בתוך המתקין
בסקריפט, אם המשתמש לא יודע את `COMPANY_ID`, הוא לוחץ Enter במקום הערך — והסקריפט מבקש את ה-token, פונה ל-edge function חדש קטן (`list-companies-for-agent`) שמחזיר רשימת `id+שם`, מציג רשימה ובוחרים מספר. זה מבטל לתמיד את הצורך לבקש ממני להוציא ID מהדאטהבייס.

### 4. עדכון README קצר ומדויק
ה-README החדש יראה רק שלושה צעדים:
1. `npm install`
2. `npm run setup`
3. סיום — ה-Service רץ.

### 5. הגנה מפני הריצה הריקה הנוכחית
אם ה-Service מותקן אבל `.env` חסר/ריק, ה-Agent יכתוב לוג ברור (`run "npm run setup"`) במקום להיכשל בשקט. כך גם אם `.env` נמחק בעתיד תדע מיד מה לעשות.

## איך זה פותר את "אני לא רוצה לחזור על זה"
- אין יותר עריכת קובץ ב-Notepad.
- אין יותר חיפוש `COMPANY_ID` בדאטהבייס — נבחר מרשימה.
- אין יותר התקנה ידנית של ה-Service אחרי כתיבת `.env`.
- בעתיד, אם תעביר את ה-Agent למחשב חדש: `git pull` (או העתקה), `npm install`, `npm run setup` → סיום.

## פרטים טכניים
- `setup.js` משתמש רק ב-`readline` של Node — אין תלויות חדשות.
- Edge function חדש `list-companies-for-agent`: מאומת מול `ATTENDANCE_INGEST_TOKEN` (אותו secret שכבר קיים), מחזיר `[{id, name}]`. `verify_jwt = false` ב-`config.toml`.
- אם `setup` רץ עם `--non-interactive` ועם flags, אפשר גם דרך CLI (שימושי לסקריפטים עתידיים).
