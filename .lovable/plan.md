# תוכנית — סידור טבלת הציוד ב-/assets

## הבעיה
טבלת הציוד מציגה את כל 53 הפריטים ברצף אחד. כשבוחרים קטגוריה כמו "ציוד עזר" עם 29 פריטים — כולם מופיעים בערבוביה לפי סדר הכנסה, ולא ברור איזה ציוד שייך לאיזה עובד.

## ברירת המחדל החדשה: תצוגה מקובצת
הטבלה תיפתח כברירת מחדל **בתצוגה מקובצת** עם שני מצבי קיבוץ אפשריים:

- **קיבוץ לפי קטגוריה** (ברירת מחדל ראשית) — כל קטגוריה מקבלת כותרת משנה (מחשבים ניידים · 7, ציוד עזר · 29 וכו'). בתוך כל קבוצה מיון משני לפי שם עובד ואז מזהה.
- **קיבוץ לפי עובד** — כל עובד מקבל כותרת משנה עם שמו וכמות הפריטים שלו; פריטים ללא שיוך תחת קבוצת "במלאי".
- **ללא קיבוץ** — תצוגה שטוחה רגילה, לעבודה עם מיון עמודות חופשי.

מתג בראש הטבלה (Toggle group של 3 כפתורים): "קטגוריה / עובד / ללא קיבוץ".

כשבוחרים טאב של קטגוריה ספציפית בראש העמוד — הקיבוץ עובר אוטומטית ל"עובד" (כי הקטגוריה כבר מסוננת).

## תכונות נוספות

### 1. כותרות משנה דביקות וקבוצות מתכווצות
כל כותרת קבוצה כוללת:
- שם הקבוצה (קטגוריה / שם עובד / "במלאי")
- מספר פריטים בקבוצה
- חץ קטן לכיווץ/הרחבה

הכותרת sticky כשגוללים בתוך הטבלה.

### 2. מיון לחיצה לפי כותרת עמודה
כל כותרת בטבלה (מזהה / שם פריט / קטגוריה / בעלות / סטטוס / תפוגה) ניתנת ללחיצה למיון עולה/יורד.
- מזהה ממוין "טבעי" כך ש-`EZR-0526-002` בא לפני `EZR-0526-010`.
- במצב מקובץ — המיון פועל בתוך כל קבוצה.

### 3. סינון סטטוס
Dropdown סטטוס נוסף ליד "סינון לפי עובד": הכל / בשימוש / במלאי / בתיקון / אבד.

### 4. שורת סיכום
מתחת לסינונים: "מציג X מתוך Y פריטים · בשימוש: A · במלאי: B".

## הערות טכניות
- כל הלוגיקה בצד הלקוח על המערך `filtered` הקיים — לא נדרשים שינויים ב-Supabase.
- מיון טבעי באמצעות `localeCompare(b, undefined, { numeric: true })`.
- מצב הקיבוץ והמיון נשמרים ב-state מקומי בלבד.
- קבצים מושפעים: `src/pages/Assets.tsx` בלבד.

## תוצאה
- ברירת מחדל: טבלה מקובצת לפי קטגוריה — שקט ויזואלי וקל למצוא ציוד
- מתג מהיר לעבור בין "קטגוריה / עובד / ללא קיבוץ"
- כל עמודה ניתנת למיון בלחיצה
- סינון נוסף לפי סטטוס + שורת סיכום
