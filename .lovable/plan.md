# תוכנית — דיאלוג פרטי נכס במצב צפייה

## הבעיה
לחיצה על שורת נכס בטבלה פותחת מיד את הדיאלוג במצב עריכה — כל השדות פתוחים, מקלדת זמינה, וקל בטעות לשנות ערכים. המשתמש רוצה תחילה לראות את הפרטים, ורק אם יבחר — לעבור למצב עריכה.

## ההתנהגות החדשה
דיאלוג הנכס ייפתח כברירת מחדל ב**מצב צפייה (view)**:
- כל השדות מוצגים כטקסט קריא בלבד (לא inputs).
- כפתור **"ערוך"** בולט בכותרת + בתחתית.
- היסטוריית הבעלות, פרטי הקטגוריה ומסמכים מוצגים כרגיל.
- מסמכים ניתנים להעלאה/הורדה/מחיקה גם במצב צפייה (אינם חלק מ"שמירת השינויים" של הטופס — הם נשמרים מיידית).
- הסקשן "טופס מסירת ציוד" (כפתור "הפק טופס מסירה") נשאר נגיש גם במצב צפייה.

לחיצה על "ערוך" מחליפה את הדיאלוג ל**מצב עריכה**:
- שדות הופכים ל-inputs פעילים (כפי שהוא היום).
- בתחתית: "ביטול" (חזרה לצפייה ללא שמירה) + "שמור שינויים".
- שמירה מוצלחת מחזירה את הדיאלוג למצב צפייה ומציגה toast.

בכל פעם שהדיאלוג נפתח מחדש — הוא חוזר ל-view.

## מימוש טכני
- ב-`src/components/EditAssetDialog.tsx`:
  - הוספת state `mode: "view" | "edit"` עם reset ל-"view" ב-useEffect של פתיחת asset.
  - עטיפת כל שדה inputs ב-helper שמרנדר input במצב edit וטקסט פשוט במצב view (יצרן/מודל, מספר סידורי, שם, קטגוריה, שיוך, מצב, תפוגה, הערות, custom fields).
  - `CustomFieldsRenderer` יקבל prop חדש `readOnly` שמרנדר את הערכים כטקסט.
  - כותרת הדיאלוג תכלול כפתור "ערוך" קטן בצד שמאל כשבמצב view.
  - footer דינמי: view → ["סגור", "ערוך"]; edit → ["ביטול", "שמור שינויים"].
- מסמכים (`AssetDocumentsSection`) ללא שינוי — תמיד אינטראקטיבי.

פתיחת נכס מציגה דף פרטים שקט ובטוח מטעויות, עם מעבר ברור לעריכה כשצריך.
