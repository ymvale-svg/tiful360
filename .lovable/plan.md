## תוכנית סופית — ימי הולדת בפורטל עם תמיכה בלוח עברי

### 1. סכמת DB
הוספה לטבלת `employees`:
- `birthday_calendar_preference` — enum `'gregorian' | 'hebrew'`, ברירת מחדל `'gregorian'`
- `hebrew_birth_day` — `smallint` (1–30)
- `hebrew_birth_month` — `smallint` (1–13, כולל אדר א'/ב')
- `hebrew_birth_year` — `smallint` (שנה עברית, למשל 5745)

`birth_date` הלועזי נשאר כפי שהוא. הערכים העבריים נשמרים **כמו שהוזנו** ולא נגזרים מהמרה.

### 2. ספרייה
`@hebcal/core` (~50KB) — להמרה של תאריך עברי מקורי → תאריך לועזי בשנה הנוכחית (לצרכי תצוגה והשוואה ל"היום"), וגם לפורמט גימטריה עברי.

### 3. RPC `get_company_birthdays`
יחזיר לכל עובד גם:
- `birthday_calendar_preference`
- `hebrew_birth_day`, `hebrew_birth_month`, `hebrew_birth_year`

החישוב של "האם יום ההולדת החודש/השבוע" + מיון ייעשה בצד הלקוח (כי המרה עברית→לועזית דורשת `@hebcal`).

### 4. RPC חדש `update_my_birthday_preference`
מאפשר לעובד מחובר לעדכן רק את ההעדפה והתאריך העברי של עצמו (נקבע לפי `linked_user_id = auth.uid()`).

### 5. UI — כרטיס חדש בפורטל ("הגדרות יום הולדת")
- רדיו: **לועזי** (ברירת מחדל) / **עברי**
- אם נבחר עברי → 3 שדות: יום (1–30), חודש (Select עם השמות: ניסן, אייר, … כולל אדר א'/ב'), שנה (input מספרי, למשל 5745).
- הסבר קצר: *"התאריך העברי נשמר כפי שהוזן ולא מומר מהתאריך הלועזי."*

### 6. באנר ימי ההולדת — שינויים
- מיון לפי יום-בחודש (לועזי בפועל) עולה.
- תצוגה:
  - לועזי: `4 ביוני`
  - עברי: גימטריה — למשל **ט"ו ניסן תשמ"ה** (יום + חודש + שנה, ללא אימוג'י, ללא תווית "(עברי)").
- אם יום ההולדת היום → `🎉 היום!` במקום התאריך.
- אם מחר → `מחר`.
- ה־chip יקבל מעט יותר ריווח (`gap-2`).

### 7. עריכת עובד (אדמין) ב־`EditEmployeeDialog`
הוספת אותם שדות (העדפה + תאריך עברי) כדי שאדמין יוכל למלא עבור עובד שלא נכנס לפורטל.

### פירוט טכני
- פורמט גימטריה: `new HDate(d, m, y).renderGematriya()` של `@hebcal/core` מחזיר בדיוק `ט"ו ניסן תשמ"ה`.
- חישוב התאריך הלועזי השנה לעובד "עברי":
  ```ts
  const today = new HDate(new Date());
  let candidate = new HDate(emp.hebrew_birth_day, emp.hebrew_birth_month, today.getFullYear());
  // אם כבר עבר השנה, נציג רק אם עדיין בטווח הרלוונטי
  const gregThisYear = candidate.greg();
  ```
- טיפול בחודש אדר בשנה פשוטה: אם המקור הוא אדר א' או אדר ב' ושנת היעד פשוטה — נפול חזרה ל"אדר" (`@hebcal` מטפל באמצעות `HDate` יציב).
- אינדקס: יתעדכן `mem://index.md` עם רשומה ל־`mem://features/birthdays-hebrew` המתעדת את הכלל "תאריך עברי נשמר מקורי, לא מומר; פורמט גימטריה; ללא אימוג'י".
