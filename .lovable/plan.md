## הבנת הבקשה

לפי הצילום שצירפת, "תת קטגוריה" = פריט בודד בתוך קטגוריה (למשל "כרטיס נוכחות EZR-0526-001"). כיום זה מוצג כרשימת שורות. נשנה לתצוגת אייקונים, וגם נשפר את אייקוני הקטגוריות עצמם.

## מה נשנה

### 1. רמה 2 — רשימת פריטים בקטגוריה (`CategoryAssetsList.tsx`)

החלפת תצוגת הרשימה בתצוגת **גריד של כרטיסי אייקון** (עם אפשרות החלפה לרשימה למי שמעדיף):

- כל פריט יוצג ככרטיס מרובע עם:
  - אייקון של הקטגוריה (יורש מ-`getCategoryIcon`) במעגל צבעוני
  - שם הפריט + מזהה (`asset_code`)
  - שורה תחתית: שם בעלים (אם מוקצה) או "במלאי"
  - תג סטטוס קטן (בשימוש / במלאי / בתיקון / אבד) — צבע מבדיל
  - מס׳ סידורי כשורת משנה דקה
- לחיצה תוביל לרמה 3 (פרטי הפריט + רשימת העובדים המשוייכים / מצב מלאי) — קיים כבר ב-`AssetDetailView`
- כפתור toggle "אייקונים / רשימה" בראש המסך, ברירת מחדל = אייקונים

### 2. רמה 1 — גריד הקטגוריות (`CategoriesGrid.tsx`)

- **הגדלת האייקונים**: עיגול 16-20 (מ-12 כיום), אייקון `w-9 h-9` (מ-`w-6`), כרטיסים גדולים יותר (פחות עמודות: עד 4 ב-xl)
- **צבעים מבדילים**: כל קטגוריה תקבל גוון משלה לפי האייקון/שם — palette של ~10 גוונים (כחול, סגול, ורוד, ירוק, כתום, ענברי, טורקיז, אינדיגו, אדום, צהוב), נקבע דטרמיניסטית מ-hash של שם הקטגוריה כדי שיישאר יציב

### 3. מילון אייקונים (`categoryIcons.ts`)

- הוספת map צבעים מקביל: לכל מפתח (ביטוח / רכב / לפטופ וכו׳) — צבע ברירת מחדל מתאים סמנטית (ביטוח=ירוק, רכב=כתום, מסמכים=ענבר, מפתח=צהוב וכו׳)
- פונקציה חדשה `getCategoryColor(name)` שמחזירה זוג token-ים (`bg`, `text`) מתוך palette קבוע של Tailwind. אם השם לא מזוהה — fallback ל-hash של השם → גוון מהפלטה

## פירוט טכני

```text
src/
  lib/
    categoryIcons.ts          ← מוסיפים getCategoryColor() + COLOR_PALETTE
  components/assets/
    CategoriesGrid.tsx        ← אייקונים גדולים + שימוש ב-getCategoryColor
    CategoryAssetsList.tsx    ← תצוגת גריד + toggle לרשימה
```

פלטת צבעים (Tailwind classes, עובדת גם ב-dark mode):
```ts
// דוגמה
{ bg: "bg-blue-500/15",   text: "text-blue-600 dark:text-blue-400"   }
{ bg: "bg-emerald-500/15",text: "text-emerald-600 dark:text-emerald-400" }
// ... 10 גוונים
```

לא נוגעים בסכמה ולא במסד נתונים — טהור frontend.

## מה לא משתנה

- מבנה ה-3 רמות (קטגוריות → פריטים → פרטי פריט)
- ה-routing דרך URL params
- הדיאלוגים הקיימים (הוספה / יבוא / עריכה)
- פרטי הפריט ברמה 3 כבר מציגים בעלים + היסטוריה — נשארים כמו שהם
